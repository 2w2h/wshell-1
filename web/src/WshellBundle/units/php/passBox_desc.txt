<p>Этот юнит хранит пароли. Помимо очевидного способа использования он также может быть полезен
при составлении цепочек, требующих авторизации на сторонних сайтах. Доверять ему или нет - решать, конечно же, вам.</p>
<p>Что касается использования <b>парольных менеджеров</b>(ПМ) вообще - да, они необходимы. Все ваши пароли должны быть разными и сложными, если можете хранить их в голове - поздравляю с отличной памятью. В противном случае обязательно используйте ПМ.</p>

<h3>Как?</h3>
<p>Пароли хранятся на сервере в зашифрованном виде, само шифрование основано на алгоритме Blowfish. В качестве ключа используется общий мастер пароль, которые нужно запомнить. <b>В случае потери мастер пароля все остальные данные восстановить невозможно.</b></p>
<p>Хранимые пароли делятся на 2 категории - без контекста(т.е. собственно пароли) и с контекстом - минимально необходимой информацией для автоматической работы с парольной аутентификацией.</p>
<p>При загрузке и дешифровке паролей с контекстом генерируются ссылки
для логина в один клик. Тут есть два важных момента:</p>
<ul>
<li>В ссылках обязательно используется https, а не http. Таким образом
браузер сначала устанавливает безопасное соединение с сервером,
и только затем передаются данные. В этом не сложно убедиться, просмотрев
обмен данными через любой сетевой анализатор, например Wireshark.</li>
<li>Ссылки открываются без реферера, что позволяет обойти ограничение
на кросс-доменные переходы, устанавливаемое некоторыми сайтами.</li>
</ul>
<p>Также вместе с каждым паролем сохраняется уникальный вектор инициализации. Суть его в том, что даже одинаковые пароли в зашифрованном виде будут гарантировано разными.</p>

<h3>Режимы</h3>

<ul>
<li><i>GUI</i>: save (сохранить пароли) и load (загрузить пароли).</li>
<li><i>CLI</i>: только извлечение данных с контекстом.</li>
<li><i>Embed</i>: отсутствует.</li>
</ul>
